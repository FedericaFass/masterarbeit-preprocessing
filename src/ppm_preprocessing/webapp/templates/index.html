Configure & Compare

Set preprocessing options, then compare your configuration against a clean baseline

Preprocessing Options
Outlier Detection (IQR) Remove cases with extreme durations (IQR method) — works for all task types.
Filter Rare Classes Remove activity classes with fewer than N training samples.
Time Window Filter Only use cases that started on or after a chosen date. Older cases may reflect outdated process versions — filtering them focuses training on current behaviour.
Rare Variant Filter Remove cases whose activity sequence occurs fewer than N times. Rare variants are noise that hurts generalisation.
min
Filter Consecutive Duplicates Remove repeated consecutive events with the same activity within a case (e.g. A→A→B becomes A→B). Eliminates logging artifacts.
Impute Missing Attributes Fill missing case attribute values with column median (numeric) or mode (categorical). Ensures complete feature vectors for all encoders.
Train / Val / Test Split Percentage of cases used for each set. Test is computed automatically.
% % 15%
Test set must be ≥ 5%. Adjust train or val.
Temporal Ordering Sort cases by start time before splitting (oldest → train, newest → test) instead of random assignment. Prevents temporal data leakage.
